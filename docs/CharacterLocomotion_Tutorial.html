Walk & Run Locomotion
with Blend Spaces

How locomotion works here

Every character has an animation controller — a small state machine that picks clips and cross-fades between them. For walk/run you don't need many states: you need one state that is a blend space, driven by a single Speed parameter.

The four pieces
  • States — each names a clip (or a blend space) with loop/speed. The controller is in exactly one at a time.
  • Parameters — named values that drive the graph: Speed (Float), plus optional Direction, Bools, and Triggers.
  • Blend space — a state that holds several clips positioned along a parameter axis and blends continuously between them (idle → walk → run as Speed rises).
  • Transitions — rules that move between states when a parameter crosses a threshold (used for jump/fall/etc., not needed for basic walk↔run — the blend space handles that).
The key shortcut The player controller and AI automatically feed the parameters Speed, Direction, Acceleration, Deceleration, TurnRate and VerticalSpeed into the character's controller every frame. So if you name your blend parameter exactly Speed, walking and running in the level drives the blend with zero scripting. Match those names and it just works.

1 · Load the character & confirm clips

  1. Open the Character Editor panel.
  2. In the Mesh component, assign your Skeletal Model (the rig with your idle/walk/run animations) and a Material.
  3. Switch to the Animation component. The clip dropdowns should now list your imported clips (if they say "Load a skeletal model to choose clips", the model didn't load — fix the Mesh first).
Prereqs Your model must be skeletal and its idle/walk/run clips imported and named. Jot the exact clip names — you'll pick them per blend sample. Mixamo/glTF rigs (Y-up) drop in cleanly.

2 · Create the locomotion blend space (the fast path)

In the Animation component, under States, click Create 1D Locomotion.

Animation ▸ States ▸ Create 1D Locomotion

This one button does the whole setup for you:

  • Adds a Speed Float parameter (if you don't already have one).
  • Adds a state named Locomotion whose Blend Parameter is Speed.
  • Seeds three blend samples: clip 0 at axis 0, clip 1 at axis 2, clip 2 at axis 6 — i.e. idle → walk → run.
What "axis value" means A sample's Axis Value is the Speed at which that clip plays purely. Between samples the controller blends. So idle=0, walk=2, run=6 means: at Speed 0 you get pure idle, at 2 pure walk, at 6 pure run, and anything in between is a weighted blend. Match these numbers to your character's real movement speeds (next steps) and foot-sliding disappears.

3 · Assign clips & tune the samples

Expand the Locomotion state. You'll see it already has Use Blend Space 1D ticked and a Blend Parameter of Speed. For each sample:

  1. Animation — pick the clip (Idle for sample 1, Walk for 2, Run for 3).
  2. Axis Value — the Speed that clip represents. Start with 0 / 2 / 6 and refine to match your walk/run speeds from the player controller (Step 7).
  3. Tick Synchronize Animation Cycles so the walk and run cycles keep their feet in phase while blending — this is the single biggest anti-foot-slide setting.

Use Add Blend Sample to add more stops (e.g. a jog between walk and run), or Remove to drop one. Order doesn't matter — the controller sorts by Axis Value.

Preview it live The Character Editor preview shows the model. Scrub the Speed value in the preview's parameter controls and watch idle blend into walk into run. This is the fastest way to dial in the Axis Values before you ever enter the level.

4 · Parameters (and what drives them)

Under Parameters you'll have Speed (Float) from Step 2. Each parameter has a Name, a Type (Float / Bool / Trigger) and a Default. Add more with Add Parameter.

You rarely set these by hand at runtime — the engine publishes movement into a fixed set of names every frame. Name your parameters to match and they're driven for free:

ParameterTypeFed automatically with…
SpeedFloathorizontal movement speed (the walk/run driver)
DirectionFloatmove direction in degrees (for 2D strafe blends)
Acceleration / DecelerationFloatrate of speed change (starts/stops)
TurnRateFloatyaw turn rate (lean/turn animations)
VerticalSpeedFloatvertical velocity (jump/fall)

Fed by the standalone player, the editor's Play mode, and AI agents alike — so the same graph works whether the character is player-controlled or an enemy.

5 · (Optional) 2D directional blend space

If your character strafes or locks onto a target, use a 2D blend space (Speed × Direction) so it plays the right forward/strafe/back clip while moving. The one-click setup is:

Animation ▸ States ▸ Create 2D Directional Locomotion

It adds Speed, Direction (and a few helper Float/Bool params) and a Directional Locomotion state with 2D Directional Blend Space on. Samples carry both an Axis Value (Speed) and a Direction (deg): e.g. idle (0,0°), walk fwd (2,0°), run fwd (6,0°), strafe-left (2,−90°), strafe-right (2,90°), back (2,180°). Assign your directional clips to each.

When to bother Only use 2D if the character actually moves sideways independent of facing (twin-stick, lock-on, cover shooter). For a standard follow-cam character that always faces its move direction, the 1D blend space is simpler and looks great.

6 · Add states & transitions (jump / fall / etc.)

Basic walk↔run needs no transitions — the blend space covers it. You add states and transitions for things outside the ground locomotion, like jumping or falling.

Add the states

Click Add State for each: e.g. Jump (jump clip, Loop off) and Fall (fall/loop clip). Keep Locomotion as your base ground state.

Wire the transitions

Under Transitions, click Add Transition and set the fields:

Then a return: Fall → Locomotion when VerticalSpeed >= 0 (or an IsGrounded Bool == true), with a short Fade.

7 · Put the character in the level

  1. Save the character, then get it into the scene: Character Editor ▸ Add to Scene, or select an object and Apply to Selected, or drag the saved .3dgcharacter from the Content browser.
  2. Add a Create ▸ Gameplay ▸ Player Start and enable its Player Controller so you can drive the character.
  3. In the controller, set Walk Speed and Run Speed to match your blend Axis Values — e.g. walk ≈ 2, run ≈ 6. This alignment is what makes the feet match the ground.
Test Press P to Play and move. Walking should show the walk blend; holding run (your sprint key) should ramp Speed up and blend into the run. Because Speed is fed automatically, there's nothing else to hook up.

8 · Tune it (kill foot-sliding & pops)

SymptomFix
Feet slide while walking/runningMake each sample's Axis Value equal the character's actual speed at that clip, and match the controller's Walk/Run speeds to them. Enable Synchronize Animation Cycles.
Run looks too slow / fastNudge the run sample's Axis Value, or set the state/clip Speed multiplier slightly (e.g. 1.1).
Harsh pop between walk and runThey should blend, not transition — confirm they're samples in one blend space, not separate states joined by a transition.
Snappy jump/fall changesIncrease the transition Fade (0.1–0.2s).
Character stuck in idleSpeed isn't rising — confirm the parameter is named exactly Speed and the object has a player controller (or nav agent) driving it.
T-pose in the levelModel not marked skeletal, or clip names don't match. Re-pick clips in the state; confirm the Mesh is a Skeletal Model.

Appendix · Quick reference

Minimal walk/run recipe

  1. Mesh: assign Skeletal Model + Material.
  2. Animation ▸ States ▸ Create 1D Locomotion.
  3. Set the three samples to Idle / Walk / Run; Axis Values 0 / 2 / 6; tick Synchronize Animation Cycles.
  4. Scrub Speed in the preview to verify the blend.
  5. Add to Scene → Player Start + Player Controller (Walk 2, Run 6) → Play.

Auto-driven parameters

Speed, Direction, Acceleration, Deceleration, TurnRate, VerticalSpeed — set each frame by the player controller and AI. Name your graph parameters to match and no scripting is required.
