What you'll build

A compact 3rd-person arena: you spawn in a walled play space, dodge a patrolling enemy that chases on sight, grab an objective pickup, and reach the exit to win — lose if your health hits zero.

The gameplay loop
  1. Spawn as a 3rd-person character in a walled arena.
  2. Explore to find the glowing objective pickup.
  3. Avoid an AI enemy that patrols and chases when it sees you (it damages you on contact).
  4. Collect the objective, then reach the exit trigger.
  5. Win screen on exit; lose screen if health reaches zero.

0 · Before you start

Build the editor and player once, from a clean configure. The root project is trimmed to just the game targets (engine, game, editor, player), so this is quick.

cmake -S . -B build -DGAMEENGINE_STATIC_RUNTIME=ON -DCMAKE_BUILD_TYPE=Release
cmake --build build --config Release
  • GAMEENGINE_STATIC_RUNTIME=ON makes the eventual player build run on any Windows PC with no redistributable.
  • Launch build/bin/Release/3DGEditor.exe (or your build's bin path).
Controls in the editor viewport: right-mouse + WASD to fly the camera · W/E/R switch gizmo to move/rotate/scale · F frame the selected object · P toggle Play mode.

1 · Create the game project

Give the demo its own project so its assets and scenes are self-contained and packageable.

File ▸ Project ▸ Choose Location… → pick a folder in the native dialog · type a Name (e.g. ArenaDemo) · Create Project.

The editor scaffolds <location>/ArenaDemo/Content/Scenes/, drops you into a default scene, and remembers the project next launch. Everything you author now lives under that Content folder — which is exactly what gets packaged later.

Tip Keep asset references relative (drag assets from the Content browser rather than typing absolute paths). Relative paths survive being copied to another machine at package time.

2 · Block out the arena

Start with a floor and four walls — pure gray-box.

Floor

Create ▸ Physics ▸ Static Floor — a large static, collidable ground plane.

Alternatively Create ▸ Terrain for a sculpted ground (raise/lower with sculpt mode, paint a grass/rock material), but a Static Floor is faster for a first pass and gives clean collision.

Walls

  1. Create ▸ Cube. In the Inspector, scale it long and tall (e.g. scale ≈ 20 × 3 × 1).
  2. Move it to one edge of the floor with the move gizmo (W).
  3. Duplicate (Ctrl+D) for the opposite wall; rotate/duplicate again for the other two sides.
  4. Make sure each wall has a Collider enabled in the Inspector so the player can't walk through it.
Try it Aim for an arena roughly 20×20 units — big enough to run around, small enough that the enemy is a constant threat. Frame it all with F to check proportions.

3 · Light the scene

Add a key light and set the environment so the arena reads clearly.

Create ▸ Directional Light — this is your sun. Rotate it in the Inspector so shadows fall at a readable angle (avoid straight-down).
  • Open World / Environment settings and set the time of day — the day-night cycle drives sky colour, ambient light and the sun direction together.
  • Nudge ambient up slightly so shadowed areas aren't pure black.
  • The procedural sky and image-based lighting update automatically from the environment.
Readability For an action arena, a bright mid-morning or late-afternoon setting keeps the enemy and pickup easy to spot. Save atmospheric dusk for exploration games.

4 · Player start & the 3rd-person camera

Place where the player spawns and dial in the third-person feel.

Create ▸ Gameplay ▸ Player Start — position it near one corner of the arena.

Select the Player Start and open its Player Controller settings in the Inspector. For a 3rd-person action feel:

  • Turn first person off (3rd-person orbit camera).
  • Camera distance ≈ 5, camera target height ≈ 1 (frames the character nicely).
  • Walk speed ≈ 4, run speed ≈ 7, jump ≈ 5.
  • Leave camera collision on so the camera doesn't clip through walls.
  • Optional: enable shoulder camera / lock-on for a more combat-forward feel.
Try it Hit P to Play. You should spawn and move with WASD, the camera orbiting behind you (hold/right-mouse to look, per your control scheme). Press P again to stop. You now have a playable character — everything after this is the game around them.

5 · Give the player a body (animated character)

Swap the capsule for a rigged, animated hero. This is optional for a gray-box, but it's what makes the demo feel like a game.

  1. Open the Character Editor panel.
  2. Assign a skinned model (an .fbx/.gltf with animations) and a material from the Content browser.
  3. Under animation, enable locomotion and pick the idle, walk and run clips. Set the walk-at and run-at speed thresholds so the animation blends by movement speed.
  4. Set the capsule collider to roughly wrap the model.
  5. Apply the character to your player so it drives the controller.
How locomotion works The animation graph auto-selects idle/walk/run from a single Speed parameter and cross-fades between them — you just supply the three clips and the two thresholds. For attacks or hit-reactions later, add a one-shot action layered on top (e.g. an upper-body swing while the legs keep running).

6 · The objective pickup

Place the thing the player must collect.

Create ▸ Gameplay ▸ Pickup — position it across the arena from the spawn.
  • In the Inspector, set what the pickup grants (e.g. score, or set a flag your win check reads).
  • Give it an obvious look — a bright emissive material or a small particle glow so it reads as "grab me".
  • Optionally attach a pickup sound (see Step 11).
Try it Play and walk into the pickup — it should be consumed and update your score/flag. That flag is what the exit will check in Step 9.

7 · The enemy AI

Add a patrolling enemy that chases you on sight. This needs a nav mesh, a nav agent, and a behavior.

7a · Bake a nav mesh

Create ▸ Navigation ▸ Nav Mesh Bounds Volume — scale it to enclose the whole arena floor. Then bake the editor nav mesh (Navigation / AI tools).

The bake produces the walkable surface the agent routes over. Re-bake whenever you move walls.

7b · The agent

  1. Create the enemy body (a capsule, or a second animated character).
  2. In the Inspector, enable Nav Agent: set speed, reach radius, vision range and vision angle (how far/wide it can see you).
  3. Set its team different from the player and enable auto-target so it acquires the player.

7c · The behavior

Give it a brain: either the built-in patrol → chase → search behavior, or author one in the Behavior Graph panel (a behavior tree: patrol waypoints; when the player enters vision, switch to chase; lose sight, search then return). Assign the behavior asset to the agent.

7d · Make contact hurt

Attach a Damage Zone to the enemy (or a script) so touching the player reduces Health. That's the threat that makes the pickup run tense.

Debug it Turn on the AI debug and navigation preview overlays while playing to see the agent's path, vision cone and current state. If it stands still, the nav mesh probably doesn't cover its position — re-bake.

8 · Hazards & moving parts (optional flavour)

A couple of these turn a bare arena into a small level. Add only what serves the loop.

Create ▸ Gameplay ▸Use in the arena
Damage ZoneA lava strip / spike patch the player must route around
Moving PlatformA patrolling platform to cross a gap to the exit
DoorGate the exit until the pickup is collected
Trigger VolumeFire events: open a door, start a camera move, spawn the enemy
Keep it tight One hazard and one moving part is plenty for a demo. Every extra prop is another thing to test and package.

9 · The exit & win/lose conditions

Win

Create ▸ Physics ▸ Trigger Volume — place it at the exit. Configure its trigger action (or a small script on it) to fire "win" only if the pickup flag is set.
  • Optionally gate it behind a Door that opens when the pickup is collected, so the exit is physically blocked until the objective is done.
  • On win, trigger a short camera sequence and show the win HUD (Step 10).

Lose

The player has a Health component (from the player controller / character setup). When the enemy's Damage Zone or a hazard drains it to zero, show the lose HUD. Wire "restart" to reload the scene.

Try it Play the full loop: dodge the enemy → grab the pickup → reach the exit → win screen. Then deliberately let the enemy catch you → lose screen. If both fire correctly, your game is done — the rest is presentation.

10 · HUD

Give the player feedback with the HUD Editor panel.

  • A health bar bound to the player's Health value.
  • A score / objective readout ("Objective: 0/1" → "1/1 — reach the exit!").
  • Full-screen "YOU WIN" and "YOU DIED — press R to retry" panels, hidden until triggered.

Save the .hud into your project's Content and reference it from the scene so it loads at runtime. HUD text can now use a TrueType font — load one and set it so the UI matches your game's look.

Binding values HUD widgets read named numeric/text values (like health or score) the gameplay side publishes each frame, so the bar and counters update live without custom drawing code.

11 · Audio

Sound is the cheapest polish that most raises perceived quality.

  • Ambient / music bed on the Music bus (looping).
  • Pickup SFX on the SFX bus, fired when the objective is collected.
  • Enemy / hazard cue so damage has a sound.
  • Make the enemy and pickup sounds spatial so they pan and fade with distance — great for sensing the enemy off-screen.
Mix Set per-bus volumes so music sits under SFX. Even three sounds (music, pickup, hit) transform the feel.

12 · Camera polish (optional)

Use the Camera Director to add moments:

  • A short intro sequence panning across the arena on start.
  • A camera zone that pulls back when the enemy engages, then returns.
  • A win sequence that frames the exit as you leave.

Sequences can lock input and be skippable — keep intros short and skippable so they don't annoy on replay.

13 · Playtest & iterate

  • Play (P) and run the whole loop repeatedly. Fix the thing that breaks the flow first.
  • Use the gameplay debug, physics event, and AI overlays to see triggers firing, collisions, and the agent's state.
  • Tune numbers, not systems: enemy speed & vision, player speed, pickup placement, arena size.
  • Save often (File ▸ Save) and keep a backup copy of the project folder.
Stability If anything crashes or an asset fails to load, fix it now — a demo that crashes on someone else's machine scores nothing. Watch the log panel for asset warnings.

14 · Export & package a standalone build

Turn the scene into a self-contained, uploadable build.

  1. In the editor: File ▸ Export Runtime to write the runtime scene into your project's Content.
  2. Package it with the project's helper script (from a shell with cmake on PATH):
./package_game.ps1 `
    -ContentDir "D:\path\to\ArenaDemo\Content" `
    -SceneRelative "Content/Scenes/arena.runtime.scene" `
    -GameName "ArenaDemo"

It builds Release + static runtime, stages player.exe + player.cfg + your content beside it, and produces dist/ArenaDemo.zip.

Verify Extract the zip to a fresh folder and double-click player.exe. Confirm the on-screen "Asset errors" reads 0 and the game plays. If assets are missing, re-run the packager with -StageContentsOnly and adjust -SceneRelative. Test on a second PC if you can.

All third-party libraries are statically linked, so there are no DLLs to ship; the static runtime option means no Visual C++ redistributable is required on the target machine.

Troubleshooting

SymptomLikely cause & fix
Player falls through the floorFloor/wall collider not enabled, or player capsule mis-sized. Enable colliders; check capsule radius/height.
Enemy stands stillNav mesh doesn't cover its spot, or no target. Re-bake the nav mesh after moving geometry; enable auto-target; check vision range.
Animation doesn't playWrong clip names / thresholds, or the model isn't marked skeletal. Re-pick idle/walk/run in the Character Editor; confirm walk-at < run-at.
Camera clips through wallsEnable camera collision in the player controller settings.
Win fires without the pickupExit trigger isn't checking the pickup flag — gate the win on the collected flag, or block the exit with a Door until collected.
Packaged build: "Asset errors > 0"Asset paths don't line up beside the exe. Ensure relative paths; try -StageContentsOnly. Avoid absolute asset roots.
Packaged build won't launch elsewhereRebuild with -DGAMEENGINE_STATIC_RUNTIME=ON so no redistributable is needed.

Two checkpoints that matter most

If you hit both of these, you have a shippable demo — everything else is polish:

  • Playable checkpoint (Step 4): you can spawn and move in the arena.
  • Full-loop checkpoint (Step 9): pickup → exit → win, and death → lose, both fire correctly.

Built with 3DGEngine — a from-scratch C++17 / OpenGL engine (PBR rendering, ECS, physics, skeletal animation, AI & navigation, terrain/water/grass, audio, a full ImGui editor, and a standalone runtime player). This tutorial targets a jam-scale demo; scale the same steps up for a larger game.
